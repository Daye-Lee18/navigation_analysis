# VOC1: ETA 오류 분석 보고서
**현대오토에버 2026 상반기 과제 테스트**

---

## 1. 과제 문제 해석

### 1-1. 불만 내용 (VOC)

> 목적지까지 **60분** 걸릴 것으로 내비게이션이 안내했으나,  
> 특정 교차로에서 예상보다 **20분(30%) 초과** 지연되어 **80분**에 도착함.

### 1-2. 데이터 문제로 변환

| VOC 표현 | 데이터 문제로 변환 |
|----------|------------------|
| 60분 ETA 예측 | 링크별 예측 속도 기반 통행시간 합산의 오류 |
| 특정 교차로에서 지연 | 특정 link_id 구간에서 실제 속도 < 예측 속도 |
| 30% 초과 오차 | `navi_ETA오차율 = (실제통행시간 - 예측통행시간) / 예측통행시간 ≥ 0.30` |

**핵심 질문:**  
> 어떤 링크·시간대에서 내비게이션 ETA가 실제보다 크게 낮게 예측되는가? 그 원인은 무엇인가?

---

## 2. 가설 설정 (Hypothesis)

ETA 오류의 원인을 4가지 가설로 설정

| # | 가설 | 내용 |
|---|------|------|
| H1 | 교차로 대기시간 미반영 | 교차로에서의 신호 대기 시간이 ETA 계산에 포함되지 않음 |
| H2 | 실시간 교통 데이터 지연 | 내비게이션이 사용하는 교통 데이터가 실시간을 반영하지 못함 |
| H3 | 사고/이벤트 반영 실패 | 사고·공사·통제 등 돌발 이벤트가 ETA 재계산에 반영되지 않음 |
| H4 | ETA 모델이 평균 기반 | 평균 속도로 ETA를 계산해 분산이 높은 피크 시간대를 과소추정 |

### 3-2. ETA 계산 방식

```
# 내비게이션 예측 ETA (링크별 전체 평균 속도 기반)
예측통행시간_분 = 거리(m) / (예측속도_kmh × 1000 / 60)

# 실제 통행시간
실제통행시간_분 = 거리(m) / (실제속도_kmh × 1000 / 60)

# ETA 오차율
navi_ETA오차율 = (실제통행시간 - 예측통행시간) / 예측통행시간
```

> **설계 의도:** 내비게이션은 과거 이력의 평균 속도로 ETA를 계산한다고 가정.  
> 피크 시간대 실제 속도가 전체 평균보다 낮으면 ETA오차율 > 0 (실제가 더 오래 걸림).

### 3-3. 분석 대상

- **분석 시간대:** 평일 출근 피크 (7~9시)
- **분석 요일:** 월~금
- **분석 도로:** 주간선도로 + 보조간선도로
- **총 분석 링크:** 3,611개 (피크 시간대 데이터 보유 링크)

### H1. 교차로 대기시간 미반영

**분석:** ETA 오차 상위 10개 링크의 시간대별 평균 속도 변화 추적

**결과: 시간대별 속도 (상위 10링크 평균)**

| 시간대 | 평균 속도 | 비고 |
|--------|---------|------|
| 1~5시 | 80~82 km/h | 심야 자유통행 |
| 6시 | 71.9 km/h | 출근 시작 |
| **7시** | **50.2 km/h** | ↓ 급락 시작 |
| **8시** | **35.2 km/h** | ↓ 피크 |
| **9시** | **32.6 km/h** | ↓ 최저점 |
| 12시 | 57.9 km/h | 회복 |
| 18~19시 | 52~59 km/h | 저녁 피크 |

**핵심 수치:**  
6시(71.9 km/h) → 8시(35.2 km/h) = **피크 진입 시 45.5% 속도 급락**

**해석:**
- 심야 대비 피크 속도가 절반 이하로 떨어짐
- 이 급락이 교차로 신호 대기·진입 지체와 일치 (연속된 교차로 구간에서 더 두드러짐)
- ETA 모델이 교차로 대기시간을 별도로 모델링하지 않으면 이 급락을 예측 불가

### H4. ETA 모델이 평균 기반이라 분산 반영 못함

**분석:** 피크 시간대 속도 분포 및 변동계수 분석

**결과: 전체 피크 구간 속도 통계**

| 지표 | 값 |
|------|-----|
| 평균 속도 | 25.5 km/h |
| 표준편차 | 11.8 km/h |
| 최저 속도 | 1.7 km/h |
| 최고 속도 | 98.3 km/h |
| 평균 변동계수(CV) | **0.52** |
| 30% 초과 오차 발생 비율 | **12.6%** |

**핵심 논거:**
```
30% ETA 초과 조건:
  실제통행시간 > 1.30 × 예측통행시간
  → 실제속도 < 예측속도 / 1.30

예시: 링크 평균 속도 = 40 km/h (내비 예측 기준)
  → 실제 속도 < 30.8 km/h 이면 30% 초과 발생
  → 피크 시간대 25.5 km/h < 30.8 km/h → 오차 발생
```

**해석:**
- CV=0.52: 평균 주변으로 속도가 넓게 퍼져 있음
- 평균 기반 ETA는 "좋은 날"과 "나쁜 날"을 구분하지 못함
- 피크 시간 속도가 심하게 낮은 날 (CV의 하위 25%) → 항상 ETA 30% 초과

```
[내비게이션 ETA 오류 Root Cause]

  → 링크 통행시간만 합산, 교차로 지체 미포함

2차 원인: ETA 모델이 과거 평균 기반
  → 당일 실시간 조건(혼잡도 급변, 사고 등)을 반영하는 시간이 늦음

결과: 피크 시간대 고분산 구간에서 ETA 30% 이상 과소추정
```

## 6. AI 개선안 및 미래 방향

### 6-1. 단기 개선 (데이터/알고리즘)

| 방향 | 내용 |
|------|------|
| 교차로 delay 모델 추가 | 링크 통행시간 + 교차로 평균 대기시간(신호주기 기반) 별도 합산 |
| ETA 신뢰구간 표시 | 단일 예측값 대신 "60~75분" 범위로 불확실성 표현 |
| 실시간 AccInfo 연동 강화 | 돌발 이벤트 발생 즉시 해당 링크 재탐색 트리거 |

### 6-2. 중장기 개선 (AI 모델)

| 모델 | 내용 | 기대 효과 |
|------|------|----------|
| **LSTM / Transformer** | 시계열 교통 패턴 학습 → 당일 트렌드 반영 | 당일 혼잡 변화 포착 |
| **교차로 delay 모델** | 링크별 교차로 지체 시간을 별도 학습 | H1 해결 |
| **Ensemble ETA** | rule-based + ML hybrid → 이상치에 강건 | H4 해결 |
| **개인화 경로** | 운전자 패턴(속도 성향, 선호 도로) 반영 | UX 개선 |

### 6-3. 미래 경로탐색 서비스 지향점

- **불확실성 인지 내비:** "지금 출발하면 60분 (±15분)" — 분산 정보 제공
- **사전 경고 시스템:** 피크 시간대 ETA 신뢰도 낮을 때 사용자에게 알림
- **실시간 재탐색:** AccInfo 이벤트 발생 즉시 대체 경로 자동 제시
